面向大规模中文文本分类的朴素贝叶斯并行spark算法

Overall process of text classification: 3 steps

  1. Corpus preprocessing: word segmentation, stop words removal, frequency statistics, feature selection
  2. Text representation: text -> new form ->computor
  3. Text classification based on ml algorithm

Corpus preprocessing

Word segmentation

Chinese word don’t have continuous space between words.
Segmenting text according to the meaning of words and separating words by space.

Stop words removal

Segmented words have a large number of function words,such as 的，吗，在. ->stop words.

A stop words list is needed to filter out stop words.

Storage space can be saved, dimension of feature space can be reduced, accelerate

Word frequency statistics

Count how many times a given word appears in a given document.

Feature selection

Select a feature subset from all features.->remove features which cannot reflect contents of documents-> reduce dimension of vector space and complexity

Text representation

In this paper, the Chinese text is represented by VSM. Each dimension of vector is composed of feature item and its weight. The weight is calculated with TFIDF.

Text classification

Determinate a classification for each text

Frequently-used text classification algorithm: naive Bayes,KNN,SVM,etc

Naive Bayes

Calculate each probability for each class -> select the most probable one as its classification result.
3 steps:

  1. Obtain the training samples with known classification results
  2. Count the class conditional probabilities with each feature
  3. independent->bayes’ theorem

Hadoop-based parallelization

PNBA->training and prediction

training

Contains 2 sequential MapReduce jobs

First MapReduce Job

Map->each Mapper receives a portion of the data block from the training TFIDF text vector,split each record into a specific key-value pair, aggregates all the vector weights

Reduce->each key-value pair aggregated globally, the model attribute vector ScoresFeatureAndLabel is obtained

Second MapReduce Job

Map->split into <class, sum of class feature vector> and <feature, sum of feature vector>, aggrefates all the features for each class and in all classes.

Reduce->globally aggregated, WeightsPerLabel and WeightsPerFeature are obtained.

Classification model -> ScoresFeatureAndLabel, WeightsPerLabel, WeightsPerFeature

prediction

Each Mapper receives partial data blocks from test TFIDF text vector

Call the classification model->Calculating and compare
Reducer merge calculation result of each Mapper, accuracy calculate

Spark-based parallelization

Also 2 phases->training and prediction

DIFFERENCE: Spark-based version focuses on the algorithm design RDD( resilient distributed datasets )

intro

Operation of RDD-> transformation and action

Transformation: creating a new RDD on the basis of present RDD

Action: perform the actual computation, and then return a simple-data-type value or output data to memory.

smoothing

When lamda=1 ->laplace smoothing

When Lamda<1 ->Lidstone smoothing

When the training set is large enough, the smoothing method not noly avoid the probability of0 but also keeps the classification result not be affected.

For calculating probability is caused by multiplying by several conditional probability values from 0 to 1, which would result in the underflow of floating numbers. So take Logarithm.

training

Training samples which have been marked categories –> set up text categorization model.

This formula shows the number of each class and the total number of each feature value in each class must be counted.

Spark supports the operation of HDFS to conduct row processing of text data.

Present training samples as key-value pairs and store them in HDFS—> manifest the the training process in the transformation process of RDD.

To count the frequency and number of each class and the frequency of each feature for each class:

  1. Read all text vectors and form RDD
  2. Conduct the map operation for RDD: map each row in the form of (label, (1, features))…label:class number, 1:document number, features:feature value vectors of certain text.
  3. Take label as key to conduct reduceByKey.(label,(N,featuresSum))->N:document number to certain label.

Parallelization of NB classifier’s training process based on Spark

  1. Input: the preprocessed training set
  2. steps:
    • Define zeroCombiner [class, (text number, conditional probability in this class of each feature)] for map data structure.

• Initialize the value of ZeroCombiner and calculate the total number and feature vector sum for local samples.

• Obtain class number C and total number of training samples N.

• For each class i do:

Obtain the number of samples in every class n, calculate class prior probability.

For each feature vector v in the class do:

calculate the denomiator of class conditional probability p and take the log.

END

  1. ouput：the training model made up of matrix theta and vector pi

Parallelization of classifier’s predicting process

Test samples are input to form the RDD

Use map function to calculate the prob of text samples for each class.

Take the calss with the mximum prob as the class mark

  3. Comparison of training time of text classification on various platforms
